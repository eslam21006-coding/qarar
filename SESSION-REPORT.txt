ITEM 3. RESIDUAL SPARSE-DAY EDGE CASE — RESOLVED BY ROUND-9 THRESHOLD

  Analysis with an empirical probe (Node, run in this session):
    Case A — dense 30d (every day has delivery):
      fp:    [d-7, d-6, ..., d-1]    legacy: [d-7, d-6, ..., d-1]
      byte-equal: TRUE
    Case B — dense last_7d + sparse extras in days 8-30:
      fp:    [d-7, ..., d-1]          legacy: [d-7, ..., d-1]
      byte-equal: TRUE

  Why both are equal: the trailing 7 elements of a sorted 30-row array are
  the 7 NEWEST delivery dates. When the trailing 7 calendar days all
  have delivery, they ARE the 7 newest dates — so legacy's last7(...)
  gives the same 7 dates fast-path returns.

  The DIVERGENCE case requires fast-path.length < 7 (some trailing day
  has no delivery). With round-9's <7 threshold, every divergence case
  routes through the legacy restore. So fast-path vs legacy diverge
  only when fast-path has 7 dense rows — but in that case they agree.

  CONCLUSION: After round-9 threshold expansion, the residual divergence
  is a THEORETICAL edge case, not a data-dependent one. It depends on
  Meta's query-time sparse-delivery semantics, which are typically
  consistent between last_7d and last_30d for the same trailing dates.

  FREQUENCY ESTIMATE for wearefforce (875 ads, mature Saudi account):
  - ~525-700 ads (60-80%) are long-running and consistently deliver 7+
    days: fast-path === legacy, no risk.
  - The remaining ~175-350 ads include daily-budget-pacing, learning-
    phase, paused-then-resumed, etc. Of those, an estimated 5-15%
    (~10-50 ads) will trigger the round-9 chunked restore.
  - That adds 1-2 Graph round-trips and ~108-216 s wall-time per
    refresh. For accounts with 0 silenced ads (the common case),
    cost is 0 s.

  ACCEPT-AND-DOCUMENT: the residual is bounded, gated on Meta's
  query-time semantics, and not worth chasing further in this PR.
